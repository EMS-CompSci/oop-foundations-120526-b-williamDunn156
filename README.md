# Concepts

Creating default **constructor**s.
How **inheritance** builds variants of classes by establishing 'is-a' relationships.<br>
How **association** builds dependencies between classes by establishing 'has-a' relationships.<br>
Purpose of **access modifier** **protected**.<br>

## Key vocabulary

| Term | Definition |
|------|---------|
| Default Constructor | A constructor without parameters |
| Inheritance | The feature where a **child class** is **derived** from a **parent class**, such that it encapsulates everything the parent class encapsulates with the option of encapsuating further data or methods.|
| Association | A dependency relationship between two classes, such that one class is contained within aother. |
| Composition | A type of association where the containing class is fully 'owned' by the containing class. |
| Aggregation | A type of association where the containing class is not fully 'owned' by the containing class. |
| Protected | The access modifier which gives scope to objects of the same family. |
| Overriding | The feature which allows for a child class to redefine a (virtual) method of the parent. |
| Virtual Method | A method which can be overridden. |
| Virtual Class | A class which contains some virtual methods. |
| Polymorphism | The implementation of different behavious of objects of related classes, achieved by overriding. |
